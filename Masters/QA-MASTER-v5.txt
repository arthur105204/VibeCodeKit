## NGUYÊN TẮC QA v5.0

### 1. TEST THEO REQUIREMENTS (không chỉ theo UI)
```
Mọi test case phải MAP với:
• REQ-IDs từ RRI Requirements Matrix
• Acceptance Criteria từ TIPs
• Deliverables từ Contract
```

### 2. TIERED APPROACH
```
Tier 1 (Bắt buộc):    Core Functionality + TIP Acceptance Criteria
Tier 2 (Recommended): Edge Cases + Responsive + RRI Scenarios
Tier 3 (Optional):    Performance + Accessibility + Security

→ Tier 1 PHẢI pass trước khi release
→ Tier 2 nên pass cho production quality
→ Tier 3 cho professional-grade products
```

### 3. EVIDENCE-BASED
```
Pass = có thể demonstrate + acceptance criteria met
Fail = có evidence chứng minh + REQ-ID affected
```

### 4. TRACEABILITY
```
Mỗi test case → map tới REQ-ID → map tới RRI question
→ Biết chính xác TẠI SAO test này tồn tại
→ Biết chính xác IMPACT khi test fail
```

## HƯỚNG DẪN TEST:

```
Với mỗi test case, báo cáo:

PASS — Hoạt động đúng expected + acceptance criteria met
FAIL — Không đúng expected (mô tả issue + REQ-ID affected)
SKIP — Không applicable hoặc chưa implement
PARTIAL — Hoạt động nhưng có issues nhỏ

FORMAT:
TIP-001 AC-1: PASS
TIP-001 AC-2: FAIL — Button không responsive, REQ-005 affected
REQ-015: PASS
T2.1: PARTIAL — Mobile OK nhưng hamburger menu hơi chậm
```

## FIX WORKFLOW:

```
CRITICAL/HIGH issues:
→ Chủ thầu tạo FIX TIP → Thợ implement → Re-test specific items
→ Hoặc: Trigger DEBUG PROTOCOL nếu phức tạp

MEDIUM/LOW issues:
→ Quick fix TIP hoặc defer cho later
→ Không block release

SAU KHI FIX:
→ Re-run failed tests ONLY
→ Confirm pass
→ Update Verify Report
→ Tier 1 = 100% pass → Ready for release
```
